# 회의록 자동화 에이전트 구축 작업 기획서

- **작성일:** 2026-07-10
- **작성 위치:** `2026-07-10/기획서_회의록자동화_에이전트.md`
- **목표:** 녹취록(txt·pdf 등)을 올리면 → 회의록을 자동 작성하고 → 팀원들에게 Gmail SMTP로 메일 발송하는 파이프라인을 하나의 에이전트로 구축한다.

## 1. 한눈에 보는 전체 흐름

```
[녹취록 업로드]                [회의록 작성]                    [메일 발송]
  transcripts/ 에            skills/meeting-minutes/          scripts/send_email.py
  txt·pdf 투입     ─────►     스킬 규칙대로 초안 생성   ─────►   Gmail SMTP 발송
                              outputs/<날짜>/ 에 저장            (본인 테스트 → 팀원)
```

- **원칙 1 — 한 번에 한 단계씩:** 아래 6단계를 순서대로 진행하며, 각 단계 완료 후 사용자 확인을 받고 다음으로 넘어간다.
- **원칙 2 — 날짜 폴더:** 작업(회의록 생성 등)을 할 때마다 실행일 기준 `YYYY-MM-DD` 폴더를 `outputs/` 아래에 새로 만들고 그날의 산출물을 저장한다.
- **원칙 3 — 스킬 기반 작성:** 회의록은 에이전트 폴더 안 `skills/meeting-minutes/` 스킬 규칙을 참고해 작성한다.

## 2. 에이전트 폴더 구조 설계

에이전트 루트를 `meeting-agent/` 로 두고 아래와 같이 구성한다.

```
meeting-agent/
├── CLAUDE.md                     # 에이전트 동작 규칙 (아래 3장)
├── README.md                     # 사용법 요약
├── .env.example                  # SMTP 환경변수 템플릿 (실제 .env는 커밋 금지)
├── .gitignore                    # .env, outputs 민감정보 제외
│
├── skills/
│   └── meeting-minutes/
│       ├── SKILL.md              # 회의록 작성 규칙·양식 (아래 4장)
│       └── template.md           # 회의록 마크다운 템플릿
│
├── scripts/
│   ├── send_email.py             # Gmail SMTP 발송 스크립트 (아래 6장)
│   └── read_transcript.py        # txt/pdf 녹취록 → 텍스트 추출 (선택)
│
├── config/
│   └── recipients.yaml           # 회의별 수신자(팀원 이메일) 관리
│
├── transcripts/                  # 입력: 업로드된 녹취록 원본
│
└── outputs/
    └── 2026-07-10/               # 작업일별 폴더 (회의록·발송로그 저장)
        ├── 회의록_<회의명>.md
        ├── 회의록_<회의명>.docx   # (선택) 배포용
        └── send_log.txt          # 발송 결과 기록
```

> 현재 작업 폴더에는 녹취록 5건(`01_…txt` ~ `05_…txt`)이 있으므로, 에이전트 구축 시 이 파일들을 `meeting-agent/transcripts/` 로 옮겨 첫 테스트 입력으로 사용한다.

## 3. CLAUDE.md 설계 (초안)

에이전트 루트에 둘 `CLAUDE.md`의 핵심 규칙 초안:

```markdown
# 회의록 자동화 에이전트

## 역할
녹취록을 입력받아 회의록을 작성하고, 지정된 팀원에게 메일로 발송한다.

## 작업 절차 (반드시 이 순서)
1. `transcripts/` 의 녹취록을 읽는다. (pdf는 scripts/read_transcript.py로 텍스트화)
2. `skills/meeting-minutes/SKILL.md` 규칙에 따라 회의록을 작성한다.
3. 실행일 기준 `outputs/YYYY-MM-DD/` 폴더를 만들고 회의록을 저장한다.
4. `config/recipients.yaml` 의 수신자에게 `scripts/send_email.py`로 발송한다.
   - 최초 발송은 반드시 본인 주소로 테스트한 뒤, 확인 후 실제 팀원에게 발송한다.

## 규칙
- 회의록에 없는 내용을 임의로 만들지 않는다. 녹취록 근거만 사용한다.
- 결정사항/할 일은 담당자와 기한을 반드시 명시한다. 불명확하면 "(미정)"으로 표기.
- 발송 전 수신자·제목·본문을 사용자에게 요약 보고하고 승인받는다.
- SMTP 자격증명은 .env에서만 읽고 로그·회의록에 출력하지 않는다.
```

## 5. 수신자(팀원 이메일) 관리

`config/recipients.yaml` 로 회의별 수신자를 관리한다.

```yaml
# 테스트용 (본인) — 최초 발송은 여기로만
test_recipient: your_email@example.com

# 실제 팀원 (Step 1에서 확정)
teams:
  홍보전략회의:
    - name: 정해성
      email: 예: haesung@example.com
    - name: 김유진
      email: 예: yujin@example.com
```

> ⚠️ 현재 실제 팀원 이메일은 미확보 상태다. **작업 절차 1단계**에서 사용자에게 받아 채운다.

## 6. 이메일 발송 설계 (Gmail SMTP)

- **방식:** `smtp.gmail.com:587` + STARTTLS, Python `smtplib`
- **인증:** Gmail **앱 비밀번호**(2단계 인증 필요). 일반 비밀번호로는 SMTP 로그인 불가.
- **자격증명 보관:** `.env` (커밋 금지)

```
# .env.example
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your_gmail@gmail.com
SMTP_APP_PASSWORD=xxxx xxxx xxxx xxxx   # Google 앱 비밀번호 16자리
SENDER_NAME=회의록 자동화
```

`scripts/send_email.py` 개요:
```
1) .env 로드 → 자격증명 확인
2) 인자: --to, --subject, --body(또는 회의록 md 파일), --attach(선택)
3) MIME 구성(제목/본문/첨부) → smtplib STARTTLS 로그인 → 발송
4) 성공/실패를 outputs/<날짜>/send_log.txt 에 기록
```

**발송 순서(안전장치):** ① 본인 주소로 테스트 발송 → ② 사용자가 수신 확인 → ③ 실제 팀원 전체 발송.

> ⚠️ 확인 필요: `pusan.ac.kr` 계정으로 보낼지, 별도 `gmail.com` 계정으로 보낼지. Google Workspace(학교 지메일)면 관리자가 SMTP/앱 비밀번호를 허용해야 한다.

## 7. 단계별 실행 계획

작업 절차에 맞춘 6단계. 각 단계는 **완료 기준**을 만족하면 사용자 확인 후 다음 단계로.

| 단계 | 내용 | 산출물 | 완료 기준 |
|------|------|--------|-----------|
| **0. 기획(현재)** | 본 기획서 작성 | `2026-07-10/기획서_….md` | 사용자 승인 |
| **1. 항목·수신자 확정** | 회의록 항목 확정 + 팀원 이메일 수집 | `recipients.yaml`, 최종 양식 | 이메일·양식 확정 |
| **2. 에이전트 구축** | 폴더/CLAUDE.md/스킬/스크립트 생성 | `meeting-agent/` 일체 | 파일 생성 완료 |
| **3. 회의록 작성** | 녹취록 읽어 회의록 생성·저장 | `outputs/<날짜>/회의록_*.md` | 양식대로 저장 |
| **4. 메일 발송** | 본인 테스트 → 실제 팀원 발송(SMTP) | `send_log.txt` | 실제 수신 확인 |
| **5. 흐름 통합·완성** | 업로드→작성→발송 원클릭 연결 | 통합 실행 절차 | 엔드투엔드 1회 성공 |

## 9. 리스크 / 주의사항

- **앱 비밀번호 필수:** 2단계 인증 미설정 시 SMTP 발송 실패. 사전 설정 필요.
- **학교 계정 정책:** Workspace 정책상 외부 SMTP·앱 비밀번호가 차단될 수 있음 → 개인 Gmail 대안 준비.
- **오발송 방지:** 실제 발송 전 항상 본인 테스트 + 수신자 목록 최종 확인.
- **개인정보:** 녹취록·이메일 등 민감정보는 `.gitignore` 처리, 로그에 자격증명 미출력.
```
